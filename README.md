## Middleware

В проекте реализованы следующие промежуточные обработчики:

1. Логирование запросов  
   Выводит HTTP-метод, путь и статус ответа

2. Кастомный заголовок  
   Добавляет заголовок:

```
X-Powered-By: ASP.NET Core Lab27
```

3. Проверка ключа доступа  
   Проверяет параметр `?key=secret`, при отсутствии возвращает 401

---

## Основные выводы

1. Minimal API  
   Позволяет быстро создавать API с минимальным количеством кода. Подходит для небольших сервисов и прототипов.

2. Middleware  
   Запрос проходит через цепочку обработчиков. Важно учитывать порядок выполнения.

3. Работа с JSON  
   ASP.NET Core автоматически сериализует объекты и использует camelCase для совместимости с JavaScript.

## Интересные факты

ASP.NET Core используется в реальных крупных проектах и высоконагруженных системах.

Примеры использования:

- Stack Overflow использует ASP.NET Core для обработки миллионов запросов в день благодаря высокой производительности платформы

## Ответы на вопросы

### 1. Minimal API и MVC

Minimal API:

- минимум кода
- нет контроллеров
- быстрый старт

MVC:

- структурированный подход
- контроллеры и представления
- подходит для крупных проектов

---

### 2. Порядок middleware

Неправильно:

```csharp
app.MapGet("/", () => "Hello");
app.Use(...);
```

Правильно:

```csharp
app.Use(...);
app.MapGet("/", () => "Hello");
```

---

### 3. Почему используется camelCase

- стандарт JavaScript
- удобство для frontend
- совместимость с библиотеками

---

### 4. HTTP-коды

- 200 OK - успешный запрос
- 401 Unauthorized - требуется авторизация
- 404 Not Found - ресурс не найден
- 403 Forbidden - доступ запрещен
- 500 Internal Server Error - ошибка сервера

---

### 5. dotnet run и dotnet watch run

| Команда          | Особенности               |
| ---------------- | ------------------------- |
| dotnet run       | обычный запуск            |
| dotnet watch run | автоматический перезапуск |
